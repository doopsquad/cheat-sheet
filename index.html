Ways to Stylize
CSS is used for UI, styling web pages essentially. There are a couple of ways to stylize, using inline styling, CSS files, or an internal stylesheet. CSS file rulesets are the most optimal. Internal stylesheets are just CSS file rulesets in the HTML file.

# > . > p/h1/a/etc.

Multiple selectors

Selectors

Position
The different ways a element can be positioned: /// static - the default value (it does not need to be specified) [Remains in flow] Relative – This value allows you to position an element relative to its default static position on the web page. [Remains in flow] Absolute – When an element’s position is set to absolute, all other elements on the page will ignore the element and act like it is not present on the page. It will be positioned relative to its closest positioned parent element [DOES NOT remain in flow] Fixed – We can fix an element to a specific position on the page (regardless of user scrolling). Accompanied by top, right, bottom, left [DOES NOT remain in flow] Sticky – The sticky value is another position value that keeps an element in the document flow as the user scrolls, but sticks to a specified position as the page is scrolled further. Uses top, right, bottom, left, and [Remains in flow]

Flexbox
a tool that greatly simplifies how to position elements. Flexbox layout consists of flex containers and flex items. Flex containers contain flex items, and any element can be a flex container. They help create responsive websites by adjusting child elements based on container size. To designate an element as a flex container, set its display property to flex or inline-flex. Flex containers have two axes: the main axis and the cross axis. The main axis positions flex items using properties like justify-content, flex-wrap, flex-grow, and flex-shrink. The cross axis positions flex items using align-items and align-content. Key properties: Flex: Changes child element behavior without starting new lines. Inline-flex: Creates inline flex containers. Justify-content: Positions items left to right with options like flex-start, flex-end, center, space-around, and space-between. Align-items: Vertically spaces flex items with options like flex-start, flex-end, center, baseline, and stretch. Flex-grow: Specifies if items should grow to fill a container, using numerical values. Flex-shrink: Opposite of flex-grow, defaults to 1. Flex-basis: Specifies item width before stretching or shrinking. Flex: Shorthand for flex-grow, flex-shrink, and flex-basis. Flex-wrap: Wraps items to the next line when necessary, with options wrap, wrap-reverse, and nowrap. Align-content: Aligns elements within a single row, with options like flex-start, flex-end, center, space-between, space-around, and stretch. Flex-direction: Switches main and cross axes with options row, row-reverse, column, and column-reverse. Flex-flow: Shorthand for flex-wrap and flex-direction.

Grid Basics
Grid containers and items function similarly to flex containers and items. display: grid creates a block-level grid, while display: inline-grid creates an inline grid. Grids initially contain only 1 column, which can be expanded using grid-template-columns and rows using grid-template-rows. grid-template-columns defines the number and width of columns, while grid-template-rows does the same for rows. Fraction unit and repeat function are useful for defining sizes and repetitions of rows/columns. minmax() sets minimum and maximum sizes for rows and columns. grid-gap and gap add space between rows and columns, with gap being the shorthand for both.
N/A
Grid Items
Grid lines start at 1 and end at 1 greater than the number of rows or columns. grid-row-start and grid-row-end combine to make single grid items span multiple rows, similar to grid-column-start and grid-column-end for columns. grid-row and grid-column are shorthand properties for these. grid-area combines all properties for a comprehensive layout definition.
N/A
Advanced Grid
Grid layout has column (block) and row (inline) axes. grid-template-area allows naming sections of the webpage for grid items. justify-items and align-items position items along the row and column axes. justify-content and align-content position the entire grid along these axes. justify-self and align-self override item alignment properties for individual items.
N/A
Implicit and Explicit Grids
The implicit grid algorithm determines default behavior for grid items beyond specified grid dimensions. grid-auto-rows and grid-auto-columns specify dimensions for implicitly added rows/columns. grid-auto-flow determines whether new elements should fill rows or columns, with an option for denser packing.
